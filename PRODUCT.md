# Product

## Register

brand

## Users

Discord bot developers, mostly TypeScript-fluent, ranging from a first-bot junior to operators running multi-million-guild deployments. They arrive skeptical: most have shipped with discord.js, been burned by its memory footprint, its manual registration scripts and its string-typed options, and have seen a hundred framework landing pages overpromise. Context: evaluating whether Seyfert is worth a migration or a first build, usually on desktop, often at night.

## Product Purpose

seyfert-web is the landing + docs site for Seyfert, a TypeScript Discord framework (tiny core, decorator-driven commands, inference end to end, official Redis/uWS adapters). The landing's job: make a tired Discord dev feel that this framework was built by people like them, prove every claim with a receipt (real code, real benchmarks, real bots), and get them to `npm i seyfert`. Success = guide visits and installs, not scroll depth.

## Brand Personality

Observatory precision, classic warmth, hidden nerd winks. Three layers: (1) the surface is a serious astronomical editorial — a raymarched Schwarzschild black hole, instrument-panel typography, hairline readouts; (2) the copy speaks with the warm, human voice of the classic site ("Powerful Discord Bots Made Simple" lineage — friendly, confident, never corporate); (3) the humor is buried where only the curious find it: hover winks, code-comment jokes, a Kirby easter egg. The joke is never the first thing you see, but it is always there.

## Anti-references

- **Generic AI slop**: magenta/cyan gradients, decorative glassmorphism, identical icon-card grids, "Temu galaxy" particle backgrounds. The cosmos here is astrophotographic (amber/copper accretion physics), never neon.
- **Corporate SaaS landing**: hero-metric template, "trusted by" logo walls, stock testimonials, funnel CTAs. Receipts replace claims; real community bots replace logos.
- **Generic doc-site template**: must never read as a default fumadocs/docusaurus shell with a hero bolted on. The landing is a designed editorial object.

## Design Principles

1. **Receipts, not promises.** Every claim ships with its proof: a number, a real snippet from the guide, a measured benchmark. If it can't be backed, it isn't said.
2. **Cards hold code; prose lives in rows.** Rounded bordered cards appear only around code (editor chrome). Everything else speaks the instrument language: hairlines, mono annotations, editorial splits with a left masthead rail.
3. **One idea per section.** The type story, the toolkit breadth, the boot comparison, the memory bill — each gets exactly one beat. Never re-sing a previous section's idea.
4. **Physics is load-bearing.** The black hole is real geodesics, the palette split is real (brand indigo/purple for UI, amber/copper only for the cosmos), the metaphors are astronomy-correct. Spectacle earns trust because it is honest.
5. **The wink is earned, never shouted.** Humor density ~1-2 per section, hidden in hovers, code comments and easter eggs. Aggressive humor stays quarantined at ?chad.

## Accessibility & Inclusion

Pragmatic WCAG AA: AA contrast for text, prefers-reduced-motion respected everywhere (the WebGL hero bails to a static fallback), visible focus states, alt/aria on interactive elements. Hover-only winks are decorative bonuses, not content, and carry no functional information. No formal audit commitment.
